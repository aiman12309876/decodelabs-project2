# The Blueprint (Wireframing) - DecodeLabs UI/UX Project 2

## Project Overview
An e-commerce brand is losing customers during checkout. The layout is cluttered and users are confused. They need a clean, logical structure for their landing page.

## Mission
Design the "Skeleton" of the application focusing only on User Flow and Hierarchy.

## Constraints
- Black, White, and Grey only
- No images or colors yet
- Focus on structure and layout

---

## 1. User Flow

### Current Problem
- Cluttered layout
- Confusing navigation
- No clear Call-to-Action
- Users abandon checkout

### Proposed Solution
1. **Homepage** → Clear navigation + featured products
2. **Category View** → Filtered product listing
3. **Product Detail** → Product info + "Add to Cart"
4. **Cart** → Review items + "Checkout"
5. **Checkout** → Shipping + Payment
6. **Confirmation** → Order success

## 6. Design Principles Applied

| Principle | Application |
|-----------|-------------|
| Visual Hierarchy | Hero section first, then categories, then products |
| Call-to-Action | Clear "Shop Now", "Add to Cart", "Checkout" buttons |
| Content Priority | Most important content at top (hero) |
| F-Pattern Layout | Important items in top-left to bottom-right flow |
| Progressive Disclosure | Show only what's needed at each step |
| Whitespace | Clean, uncluttered layout |
| Consistency | Same navigation across all pages |

---

## 7. Key Improvements

| Problem | Solution |
|---------|----------|
| Cluttered layout | Clean grid structure with whitespace |
| No clear CTA | Prominent "Shop Now" on hero |
| Confusing navigation | Simple top nav with categories |
| Checkout abandonment | 3-step checkout process |
| No trust signals | Add reviews and ratings |
| No product filtering | Category navigation |
